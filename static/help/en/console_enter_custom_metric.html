Enter custom metric

This dialog allows you to create custom CloudWatch metrics. For more information, please see the Eucalyptus CloudWatch documentation in the Eucalyptus User Guide.

To enter a custom CloudWatch metric

  1. Enter the namespace for your metric in the Namespace text box. Namespace names are strings you define when you create a metric. The names must be valid XML characters, typically containing the alphanumeric characters "0-9A-Za-z" plus "."(period), "-" (hyphen), "_" (underscore), "/" (slash), "#" (hash), and ":" (colon). AWS namespaces all follow the convention AWS/<service>, such as AWS/EC2 and AWS/ELB. A namespace name must be less than 256 characters. NOTE: There is no default namespace. You must specify a namespace for each data element you put into CloudWatch.
  2. Enter a name for the metric in the Metric name text box.
  3. Enter a dimension key and dimension value in the Dimension key and Dimension value text boxes. A dimension is a name-value pair that uniquely identifies a metric. A dimension helps you design a conceptual structure for your statistics plan. Because dimensions are part of the unique identifier for a metric, metric name, namespace, and dimension key-value pairs define unique metrics. ou specify dimensions when you create a metric with the euwatch-put-data command. Eucalyptus services that report data to CloudWatch also attach dimensions to each metric. You can use dimensions to filter result sets that CloudWatch queries return. For example, you can get statistics for a specific instance by calling euwatch-get-stats with the InstanceID dimension set to a specific instance ID.
  4. Click the Save changes button to save your custom metric.
